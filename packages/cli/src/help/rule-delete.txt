# Topic: rule delete     (CLI v{{CLI_VERSION}} / topic v1)

## Goal
Remove a rule and its associated test files from `.taskless/`. Does
not contact the Taskless API; purely a local filesystem operation.

## Preconditions
- `.taskless/` directory exists.
- The target rule file exists at `.taskless/rules/<id>.yml`.
- No auth required.

## Steps

1. **Identify the rule.** If the user named one, use it. Otherwise,
   list `.taskless/rules/` and ask which one. Confirm the user's
   intent — deletion is destructive.

2. **Invoke the CLI.** Run:
   ```
   npx @taskless/cli rule delete <id>
   ```

   The CLI removes:
   - `.taskless/rules/<id>.yml`
   - All matching `.taskless/rule-tests/<id>-*.yml` files
   - `.taskless/rule-metadata/<id>.yml` (if present)

3. **Report results.** Show the user what was deleted. If the rule
   had metadata (was API-generated), note that the rule cannot be
   re-iterated via the API path after deletion.

## Errors

`rule delete` is plain-text; it does not accept `--json` and does not
emit the standardized error envelope. The rule ID is required as a
positional argument and missing/invalid usage is rejected by citty
before the command runs.

| condition                         | behavior                                                                              |
|-----------------------------------|---------------------------------------------------------------------------------------|
| Rule file does not exist          | Prints `Error: Rule "<id>" not found in .taskless/rules/<id>.yml` and exits non-zero. |
| No rule ID provided               | citty prints usage and exits non-zero before the command body runs.                   |

## See Also

- `taskless help rule create` — make a new rule
- `taskless help check` — run remaining rules to confirm nothing broke

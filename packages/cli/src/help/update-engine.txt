Request a scaffold upgrade for the .taskless/ engine directory

Sends your project's current scaffold version to the Taskless backend,
which creates a pull request to upgrade the .taskless/ directory. The
CLI polls until the PR is created, then prints the URL and exits.

Prerequisites:
  Run `taskless auth login` to authenticate first.
  A .taskless/taskless.json config file with orgId and repositoryUrl is required.

Usage:
  taskless update-engine [options]

Options:
  -d, --dir <path>   Set working directory (default: current directory)
  --json             Output results as JSON

JSON Output:
  { "status": "current" }                                    Project is up to date
  { "status": "exists", "requestId": "...", "prUrl": "..." } PR already exists
  { "status": "open", "prUrl": "..." }                       PR created
  { "status": "merged", "prUrl": "..." }                     PR was merged
  { "status": "closed", "prUrl": "..." }                     PR was closed

Examples:
  taskless update-engine
  taskless update-engine --json
  taskless update-engine -d ./my-project
